Comparative Analysis of AI Vision Models in Numismatic Identification

Evaluating Qwen and Gemma across 101 coins: accuracy, grading consistency, and confidence calibration

David Crozier CBE n = 101 coins 20 countries 2 models evaluated
This case study presents a structured comparative evaluation of two large vision-language models, Qwen and Gemma, applied to the task of numismatic identification. One hundred and one coins spanning twenty countries and over 150 years of coinage were presented to each model independently under identical prompting conditions. The results reveal meaningful convergence on country identification only, with significant divergence on year and denomination labelling and near-total disagreement on condition grading, with important implications for the deployment of AI models in specialist classification tasks.

01 — Methodology

Experimental Design

Each coin was photographed and submitted independently to both models under identical system prompts. Models were asked to identify country of origin, denomination, year of issue, mint mark, metal composition, and condition grade using the Sheldon numismatic scale. Results were parsed field-by-field and compared for agreement across all 101 coins.

Both models were run locally on the MacBook Pro using Ollama, an open-source runtime that allows large language models to be served locally via a REST API. Ollama handles model quantisation, memory management, and GPU scheduling automatically, making it straightforward to run vision-capable models on Apple Silicon without cloud infrastructure. Each model was called from a Python script that submitted each coin photograph in turn, captured the JSON response, and logged timing and output fields to the comparison dataset.

03 — Key Findings

What the Full Dataset Tells Us

F 1

Country identification is the most reliable field at 81%. Most disagreements involved colonial or dependency attributions, for example British India versus United Kingdom, attributable to labelling ambiguity rather than genuine misidentification.

F 2

Year agreement at 59% is substantially weaker than initial sampling suggested. The initial ten-coin sample indicated 90%; the full dataset reveals this was unrepresentative. Year identification deteriorates significantly for older, worn, or partially legible coins where date digits are obscured.

F 3

Denomination agreement at 31% is the most significant failure mode. Disagreements arise from labelling convention differences (“20p” versus “20 Pence”), currency system misattributions, and genuine identification errors on ambiguous pre-decimal coins. This field cannot be trusted without a normalisation layer.

F 4

Condition grading is almost entirely divergent: only 2 of 101 coins received the same grade. Qwen applied G-4 to 87% of all coins, treating it as a near-universal default for circulated coinage. Gemma applied VF-20 to 46% of coins with considerably more variance. The two models are not using different rubrics; Qwen has effectively collapsed the grading scale to a single value.

F 5

Confidence scores are uninformative as a tiebreaker. Both models reported 95% on almost every coin. Of the 29 coins with a non-zero confidence delta, the maximum divergence was 35 points (coin_082), yet this produced no flagging behaviour in either model. Confidence calibration is absent.

F 6

Gemma is both faster and more discriminating. At 54.7 seconds average versus Qwen’s 60.3 seconds, Gemma is 9% quicker. It also applies a broader range of condition grades, making it the more useful baseline for grading applications once a common rubric is established via prompt engineering.

05 — Conclusions & Recommendations

Implications for Deployment

The full dataset substantially revises the picture presented by initial sampling. Only country identification reaches a level of reliability suitable for unsupervised automation. Year, denomination, and condition grading all require targeted intervention before either model can be deployed in a production numismatic context.

1Do not rely on either model for denomination identification without post-processing. At 31% agreement, denomination labelling is the most problematic field. A normalisation layer mapping free-text outputs to a controlled vocabulary is essential before any downstream use.
2Treat year identification as unreliable for pre-1950 coins. The majority of year disagreements cluster around older, worn coins. A rule-based confidence threshold based on estimated coin age and condition could flag these for human review automatically.
3Replace Qwen’s grading default with a rubric-anchored prompt. Qwen’s near-universal application of G-4 renders its condition grades meaningless for differentiation. An explicit Sheldon scale rubric with photographic reference points in the system prompt is required before any grading use.
4Prefer Gemma for throughput-sensitive pipelines. Gemma is 9% faster on average and produces more discriminating condition grades. It is the stronger baseline once grading prompt engineering has been applied to both models.
5Prioritise the 29 high-delta coins for manual review. These represent the experiment’s most contested attributions and should form the validation set for any subsequent prompt optimisation or model fine-tuning.
6Use country agreement as a first-pass quality gate. At 81%, country identification is sufficiently reliable to act as an initial filter. Coins where models disagree on country should be routed to human review before any further fields are processed.

Key finding

C 1

If the MacBook is already owned, local inference is cheaper than the cloud. The marginal cost of running this experiment locally is just 2.0p in electricity. The equivalent cloud API bill is approximately 6p — nearly four times higher — but still negligible in absolute terms. The hardware amortisation argument only applies if the machine was purchased solely for AI inference.

C 2

Cloud wins on total cost of ownership at this scale. Including hardware amortisation, local inference costs 51p for 101 coins versus approximately 6p in the cloud. The M5 would need to process approximately 3.1 million coin images before the hardware investment pays back against equivalent cloud API spend at current token prices.

C 3

Privacy is the decisive local advantage at any scale. Running inference locally means coin images, which may carry personal or commercial sensitivity, never leave the device. Cloud APIs transmit images to third-party servers. For a numismatic catalogue of any commercial value, this distinction matters regardless of cost.

C 4

Cloud costs scale linearly; local electricity costs are negligible at all volumes. At 10,000 coins the cloud bill reaches £5.84 while local electricity is £1.58 and local with full hardware amortisation is £50.78. For very high volumes, a dedicated local inference machine becomes the rational choice, but only well beyond the scale of this experiment.
